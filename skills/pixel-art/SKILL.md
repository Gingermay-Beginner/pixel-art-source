## 📐 画布与基本规格

```python
W, H = 64, 36      # 格子数（16:9）
S    = 12          # 每格放大倍数 → 最终 768×432 px
```

**风格定位：**
- 动物森友会（AC-inspired）——饱和但柔和、圆润
- **全程硬边像素块，禁止软渐变**（烟雾、云、水面光晕全部用硬边色块）
- 角色比例：大头Q版（Chibi），头占身高约2/3，**坐姿**（不用站姿）
- **不自作主张加功能**：未告知用户的细节不加；想加先问

**中套——直接画法（GCX锚点，中景正坐场景用）：**
```
GCY-16~-15 : 帽子（3格宽→5格宽）
GCY-14~-11 : 头（dx=-3~+3，宽7格，高4格）
  GCY-14   : 眼睛（dx=-2, +2）
  GCY-13   : 腮红（dx=-3, +3）
GCY-10~-6  : 身体（约5格高）
  GCY-10   : 手臂外展（dx=-4,-5 / +4,+5）
```

**细节规则：**
- **扣子**：2颗，颜色 GB_CHEEK（腮红色），x=5，y=12 和 y=14（间隔一格）
- **帽子绘制顺序**：身体 → 头（肤色）→ 眼/腮红/嘴 → **帽子**（最后，否则被头覆盖）
- **小啾啾**：必须在帽身上方一格（`HAT_TOP-1`），不可与帽身断开，不可在上方两格
- **帽子右偏【角色固定设定】**：帽子比头中心偏右1格（x=GCX-1~GCX+3，而非居中GCX-2~GCX+2）；每次画姜饼人必带

- **耳朵【定稿】**：2格宽（外列+内列）；顶部1格圆角 = **外列顶行不画（挖掉外角）**，顶行只留内列1格；粉色 BUN_IN 在内列（x=BCX-1/BCX+1），外列（x=BCX-2/BCX+2）为 BUN_B；粉色朝内符合兔子解剖，更自然（2026-04-24 May 确认）

- **嘴**：3格白色，y=BCY+4

### 共同规范

- **嘴型**：O 型嘴，竖向两格，下格稍浅
- **腿颜色**：与身体颜色相同（GB/BUN），不使用深色（GBD/BUND）
- **头部圆角**：fill 方块头后，必须对四个角用背景色覆盖（否则头是直角）
- **颜色常量**：功能删掉时同步删掉对应常量，不留残留
- **背景与角色对比**：背景颜色不与主体角色颜色相近，避免色相/明度同时接近

## 🎨 绘制规则

### 元素范围操作

> **"去掉/挖掉/缩小/缩短"** = **缩小 `fill`/`wrow` 的范围**，让那些格子从一开始就不属于该元素。**禁止用背景色覆盖替代**——背景色覆盖是掩盖，不是去掉。执行前先在内部算好新参数再动手。
>
> ❌ 错误做法：把格子改成背景色（那只是换色，元素范围没变，语义错误）
> ✅ 正确做法：直接缩小 `lfl(y1,y2, x1,x2)` 的参数范围

**移动元素：**
- 改坐标变量（如 CRX/CRY/TX），不重新绘制
- 旧位置必须显式清除（用背景色覆盖整个旧矩形范围），否则出现孤儿像素

**孤儿像素处理：**
- 移动后旧位置残留的格子，用绝对坐标 (x,y) 直接写 px 覆盖
- 或缩短 fill 范围，而非单独删格子

### 绘制顺序

后画的覆盖先画的，顺序错误会导致元素被遮住。

| 场景 | 顺序规则 |
|------|---------|
| 水面线 | `wrow(WL)` / `wrow(WL-1)` 必须最后画，之后再叠水面亮点和光带 |
| 球场白线 | 在 COURT_MAIN 填色之后最后画 |
| 皮划艇 | 在角色之后画（否则遮住角色）；hands/arms 在 kayak 之后重绘 |
| 水磨石地板 | 在旋转贴回操作之后最后画 |
| beachhut 树 | x=32 剪影树在屋顶修边后、墙体前；其余树统一最底层 |
| 帽子 | 在头（肤色）之后绘制 |
| 关节/手臂 | 在 draw_leg 之后画 |

**加完新元素后：** 必须往后检查有无 fill/wrow/set_px 覆盖同一坐标范围。

### 坐标约定

- **y 轴方向**：y 向下增大；**上移 = y 减小，下移 = y 增大**（改位置前默念确认）
- **向 May 报坐标**：必须直接说画面绝对格坐标（如"(12,9)"），禁止说旋转前的中间坐标
- **像素连接规则**：像素不能跳格断开，相邻格必须斜对角或上下左右相连

### 特殊元素规范

- **烟雾/云**：必须用硬边像素块，禁止任何软渐变
- **绿植外轮廓**：不规则有机形，禁止规则方形 fill
- **沙发**：无描边，只用缝线（SOFA_CAR_D）表示结构
- **礁石缝隙纹理**：不规律，沿大块礁石边缘1px暗边，自然轮廓感；禁止规律网格
- **波峰波谷**：必须连续至少4格，有渐变过渡；不能单格凸起
- **俯视贝果**：用俯视圆环（不是侧视O形）

### 水面绘制

- **WL 水线颜色**：保持蓝绿色系（约 `(75,185,195)`），不可改为暖橙
- **水截面透视结构**（从上到下）：
  ```
  WL-6 / WL-5  : 海平面（天空与水面交界）
  WL-4 ~ WL-1  : 海面反光区（倒影、波光）
  WL            : 水截面线（蓝绿色全宽）
  WL+1 以下     : 水下世界
  ```

## 🔄 旋转法画俯视角色

> **为什么用这个方法：** 角色需要横向放置时（俯视、侧卧、横构图等），如果直接在目标坐标系手动估坐标，x/y 方向互换后极易出错。正确做法是：先在正面坐标系（正坐/站立）画完整角色，再用 PIL 旋转贴入场景。**旋转的触发条件是横构图需要，不限于俯视。**

```python
from PIL import Image

def make_char_img(draw_fn, w, h, S):
    """在正面坐标系画角色，返回 RGBA Image"""
    img = Image.new('RGBA', (w*S, h*S), (0,0,0,0))
    buf = [[(0,0,0,0)]*w for _ in range(h)]
    draw_fn(buf, w, h)
    px = img.load()
    for cy in range(h):
        for cx in range(w):
            col = buf[cy][cx]
            if len(col)==4 and col[3]>0:
                for dy in range(S):
                    for dx in range(S):
                        px[cx*S+dx, cy*S+dy] = col
    return img

# 旋转并贴入场景
char_img = make_char_img(draw_fn, CW, CH, S)
rotated  = char_img.rotate(90, expand=True)   # 逆时针90°，头朝左
scene.paste(rotated, (paste_x, paste_y), rotated)
```

**PIL rotate(expand=True) 坐标映射公式：**
- rotate(90°逆时针)：原 `buf[y][x]` → 旋转后坐标 `(h-1-y, x)`
- rotate(270°逆时针)：原 `buf[y][x]` → 旋转后坐标 `(y, w-1-x)`
- rotate(180°)：原 `buf[y][x]` → 旋转后坐标 `(h-1-y, w-1-x)`

**方向换算（May 说的方向 → 正面坐标操作）：**

| May 说 | 正面坐标操作 |
|--------|------------|
| 向上移 | x 减小 |
| 向下移 | x 增大 |
| 向左移 | y 减小 |
| 向右移 | y 增大 |

**强制注释规范：** 每个 `rotate()` 旁必须有注释，说明旋转角度和关键角落映射，例如：
```python
# rotate(90逆时针): 原buf[0][0]→右上, 原buf[0][CW-1]→右下, 原buf[CH-1][0]→左上
rotated = char_img.rotate(90, expand=True)
```

**新姿势/新形象处理规则：**
- 以大套/中套为基础，只改需要变化的部分（手臂/腿/头朝向/服装），保持头身比例不变

## 🧑‍💻 代码操作

### 字符串替换规则

> 常见坑：函数体内每行有4格缩进，写 `old` 字符串时遗漏缩进导致静默替换失败。

1. **替换前先 `grep -n` 确认目标行实际内容**（包括缩进空格数）
2. `old` 字符串每行必须包含与文件完全一致的缩进
3. **多行替换用 `python3 - << 'EOF'` heredoc**，不用 `python3 -c "..."`（引号内缩进易出错）
4. **替换后必须验证**：再次 `grep` 确认新内容已写入，或运行脚本确认无报错

```bash
# 标准替换流程
grep -n "目标行关键词" pixel_xxx.py   # 先确认实际内容
python3 - << 'EOF'
code = open('pixel_xxx.py').read()
old = """    # 实际缩进内容
    完整的多行old"""
new = """    # 新内容
    完整的多行new"""
code = code.replace(old, new)
open('pixel_xxx.py','w').write(code)
print('done' if new in code else 'FAIL')
EOF
grep -n "新内容关键词" pixel_xxx.py   # 验证替换成功
```

### Git 存档规则

- **细调单个元素前先 `git commit` 存档**：

  ```bash
  git add pixel_xxx.py && git commit -m "checkpoint before <改动描述>"
  ```
- 需要回退时：`git checkout <hash> -- pixel_xxx.py`
- **定稿后 commit 归档**，写清楚版本描述

## 📐 构图原则

**大方向：韦斯安德森精神（对称、分层、舞台感）**
**具体执行：按 May 的6条规则来**

### 分层结构（标准竖向）
```
y=0~5   : 天空顶部（深色）
y=5~15  : 天空渐变区 + 远景（月亮/太阳/山）
y=15~20 : 中景（树冠/云/远山）
y=20~25 : 近景（地面/水面/角色肩膀以上）
y=25~35 : 前景/水下（角色下半身/海底）
```

### 水平三分法
- 左侧 x=0~20：左景深（树/礁石）
- 中间 x=20~44：主体区（角色+焦点元素）
- 右侧 x=44~63：右景深（树/建筑）

### 6条构图原则
1. 角色永远在水平中线附近（y≈20~26），不要太靠上或太靠下
2. 两人间距约10格（GCX≈27，BCX≈37），不要太近也不太远
3. 焦点元素放偏左或偏右1/3处，不和角色完全重叠
4. **前景暗，远景亮**（剪影效果）——树/礁石用深色，天空用渐变
5. 留白：角色上方至少留5~8格天空，不要顶满
6. 不对称的对称：整体左右对称，局部有差异

## 🛠️ 通用函数模板

```python
S = 12; W, H = 64, 36

def set_px(canvas, x, y, color):
    if 0 <= x < W and 0 <= y < H: canvas[y][x] = color

def fill(canvas, y1, y2, x1, x2, color):
    for y in range(y1, y2+1):
        for x in range(x1, x2+1): canvas[y][x] = color

def wrow(canvas, y, x1, x2, color):
    # x2 是包含的右边界，用 range(x1, x2+1)
    for x in range(x1, x2+1): canvas[y][x] = color

def blend(c1, c2, t):
    return tuple(int(a + t*(b-a)) for a,b in zip(c1,c2))
```

**放大循环完成后需要覆盖特定像素，必须直写 px（写 canvas 无效）：**
```python
for dy in range(S):
    for dx in range(S):
        px[cx*S+dx, cy*S+dy] = color
```

## 🧪 新角色/复杂元素开发流程

**隔离草图法**：遇到新角色或复杂元素时，先用单独小画布验证，确认满意再整合进场景。

```python
# 标准草图脚本结构（boar_draft.py 为例）
from PIL import Image

S = 24          # 放大倍数（草图可比场景大，便于看细节）
W, H = 40, 22   # 草图画布大小（刚好容纳角色）
BG = (178, 210, 160)

canvas = [[BG]*W for _ in range(H)]

# 在这里定义并调用 draw_xxx 函数
draw_boar(2, 2, 'right')

img = Image.new('RGB', (W*S, H*S))
px = img.load()
for y in range(H):
    for x in range(W):
        c = canvas[y][x]
        for dy in range(S):
            for dx in range(S): px[x*S+dx, y*S+dy] = c
img.save('xxx_draft.png')
```

**好处：**
- 改一个格子秒看结果，不用跑完整场景
- 改错不影响主文件
- 草图确认满意后，把 `draw_xxx` 函数复制进场景文件

**命名规范：** 草图文件命名 `xxx_draft.py / xxx_draft.png`，整合后可删除。
